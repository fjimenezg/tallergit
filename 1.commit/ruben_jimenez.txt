1. ¿como se inicializa un repositorio local?

git init

2. ¿como hago para que un directorio deje de ser controlado por git?


rm -rf .git

3. ¿Si agrego un archivo a un directorio que ya está siendo controlado por Git, ¿está siendo controlado por Git?
No de inmediato. Aunque el directorio está bajo control de Git, el archivo recién agregado no será rastreado hasta que se agregue explícitamente con git add y se haga un commit.

4. ¿Qué comando se utiliza para agregar un archivo al repositorio local?


git add nombre-del-archivo
para agregar todos los archivos nuevos y modificados
git add . 

5. ¿Cómo determino qué archivos fueron modificados?
git status

6. ¿Qué comando se utiliza para hacer un commit?

git commit -m "Mensaje descriptivo del cambio"

7. ¿que es un commit?

Un commit es como una foto del estado actual de los archivos en un repositorio Git en un momento específico. Es un registro permanente de cambios, lo que permite volver a versiones anteriores si es necesario. Cada commit tiene un mensaje que describe los cambios realizados y es identificado por un hash único.
