respuestas
1.para empezar a usar git e iniciar el repositorio local se utiliza "git init"
2.se utilisaria el "git reset" si se utiliza el "git reset --soft" se mueve el archivo al staging área
3.no como tal ya que se necesita el "git add" para darle el control del archivo
4.para agregar un archivo al repositorio local se utiliza "git add" siendo "git add -A" para agregar todo tipo e archivo
5.se utiliza "git status" para examinar los archivos modificados o los que aun no se han agregado con git add
6.se utiliza el comando homónimo "git commit -m "" "
7.un commit seria el mover un archivo del staging área al repositorio local con un nombre o comentario identificatorio para después volver a ese estado seguro si algo malo pasa en el código como un equivalente a "guardar partida" para los programadores
