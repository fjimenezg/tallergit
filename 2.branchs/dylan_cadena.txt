## Ejercicio 2

### 2.1. Respuestas
1.  Un branch (rama) en Git es una línea de desarrollo separada de la rama principaly permite trabajar en nuevas características o corregir errores sin afectar el código principal.

2. Los branches permiten desarrollar nuevas funciones, corregir errores o probar cambios sin afectar el código estable. Facilitan el trabajo en equipo, ya que diferentes desarrolladores pueden trabajar en distintas características de manera independiente.

3. Se utiliza el siguiente codigo git branch nombre_de_la_rama
   
4. Se pueden usar los codigos git checkout nombre_de_la_rama o git switch nombre_de_la_rama
 
5. Se pueden usar los codigos: 
   Si la rama no está fusionada:  git branch -D nombre_de_la_rama
   Si la rama ya fue fusionada o deseas eliminarla de manera segura: git branch -d nombre_de_la_rama
 
6. Se utiliza el codigo  git checkout -b nombre_de_la_rama

7.  Un merge (fusión) es el proceso de combinar cambios de una rama en otra. Generalmente, se usa para fusionar cambios de una rama de desarrollo con la rama principal (`main` o `master`).

8.  Primero, cambia a la rama en la que deseas fusionar los cambios: git checkout main  # O "git switch main"
   Luego, realiza el merge:  git merge nombre_de_la_rama
 
9. Un tag (etiqueta) es un marcador en la historia de Git que se usa comúnmente para identificar versiones específicas del código, como versiones de lanzamiento.

10. Se utiliza el siguiente codigo: git tag -a nombre_del_tag -m "Mensaje del tag"
