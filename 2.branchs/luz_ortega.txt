2.1. Preguntas

1. ¿Qué es un branch?

Un branch (rama) en Git es una línea independiente de desarrollo. Permite trabajar en cambios sin afectar la rama principal del proyecto.

2. ¿Por qué pueden ser útiles los branches?

Facilitan el desarrollo simultáneo de nuevas características, corrección de errores y experimentación sin alterar el código estable.

3. ¿Cómo se crea una branch?

Se usa el comando: git branch nombre_de_la_rama

4. ¿Cómo se cambia a una branch?

Se usa el comando: git checkout nombre_de_la_rama o git switch nombre_de_la_rama

5. ¿Cómo se elimina una branch?

Se usa el comando: git branch -d nombre_de_la_rama (si ya fue fusionada) o git branch -D nombre_de_la_rama (para forzar su eliminación).

6. ¿Cómo se crea una branch y se cambia a ella en un solo paso?

Se usa el comando: git checkout -b nombre_de_la_rama o git switch -c nombre_de_la_rama

7. ¿Qué es un merge?

Es el proceso de combinar los cambios de una rama en otra, generalmente integrando una rama de desarrollo en main.

8. ¿Cómo se realiza un merge?

Primero se cambia a la rama destino con git checkout main, luego se ejecuta git merge nombre_de_la_rama.

9. ¿Qué es un tag?

Un tag (etiqueta) es una referencia fija a un commit específico, usado comúnmente para marcar versiones importantes del proyecto.

10. ¿Cómo se crea un tag?

Se usa el comando: git tag -a nombre_del_tag -m "Mensaje del tag"
