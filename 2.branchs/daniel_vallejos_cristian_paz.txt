Ejercicio 2
2.1
    1.  Un branch es una rama independiente en un proyecto, donde se puede hacer cambios sin afectar la versión principal. Es útil para trabajar en nuevas funcionalidades o experimentar sin riesgos.
    2.  Son útiles porque te permiten trabajar en varias cosas a la vez sin desordenar el código principal.
    3.  git branch <nombre-de-la-rama>
    4.  git checkout <nombre-de-la-rama> ó git switch <nombre-de-la-rama>
    5.  git branch -d <nombre-de-la-rama>
    6.  git checkout -b <nombre-de-la-rama>
    7.  Es cuando se combina los cambios de una rama con otra.
    8.  Primero se cambia a la rama en la que se quiere fusionar los cambios (git checkout <rama-destino>),
        luego fusiona la otra rama ejecutando: git merge <rama-a-fusionar>
    9.  Es una etiqueta fija en la historia del repositorio, usada para marcar versiones o hitos importantes.
    10. git tag <nombre-del-tag> ó git tag -a <nombre-del-tag> -m "Mensaje del tag"
