## Ejercicio 3

### 3.1. Respuestas

1. Un conflicto en Git ocurre cuando dos ramas han modificado la misma parte de un archivo de forma diferente, y Git no puede decidir automáticamente cuál versión mantener.
    No es ni bueno ni malo, es simplemente una situación que requiere intervención manual.
2. Para minimizar conflictos en Git, sse puede sincronizar el código con `git pull` antes de hacer cambios, dividir el trabajo en archivos distintos, realizar commits pequeños y usar `git rebase` en lugar de `merge`.
